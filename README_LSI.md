# LSI — агрегационный слой Liquidity Stress Index

## Что делает слой

LSI — это финальная «крышка» над модулями M1–M5. Он не скачивает новые данные, а берёт уже подготовленные сигналы:

- M1: `data/m1/results/m1_signals.xlsx`
- M2: `data/m2/results/m2_signals.xlsx`
- M3: `data/m3/result/ofz_auctions_m3_signals.xlsx`
- M4: `data/m4/results/m4_signals.xlsx`
- M5: `data/m5/result/m5_treasury_signals.xlsx`

На выходе формируется единый индекс `LSI` от 0 до 100, статус и вклад каждого модуля.

## Почему выбран такой алгоритм

Для текущей версии выбран не градиентный бустинг, а интерпретируемая взвешенная сумма. Это проще защитить: по каждой дате видно, сколько пунктов LSI дал каждый модуль.

Формула:

```text
LSI = clip((w1*M1 + w2*M2 + w3*M3 + w5*M5) * Aggregation_Scale * Seasonal_Factor, 0, 100)
```

Базовые веса:

```text
M1 = 0.22
M2 = 0.28
M3 = 0.20
M5 = 0.30
```

M4 не суммируется как отдельный стресс-модуль. Он используется как сезонный множитель, потому что налоговый эффект уже частично отражается в M1, M2 и M5.

## Как учтена связь между модулями

1. Все модули приведены к дневной шкале.
2. M1 и M5 имеют низкую частоту, поэтому используется последнее известное значение.
3. M2 учитывается по дням аукционов.
4. M3 удерживает сигнал 7 дней после аукциона ОФЗ, потому что аукционы проходят не каждый день.
5. M4 применяется как контекст:
   - `Seasonal_Factor` усиливает общий индекс;
   - в налоговые недели веса M1/M2/M5 снижаются на 15%, чтобы не было двойного счёта налогового давления.

## Статусы

```text
0–40   — ЗЕЛЁНЫЙ
40–70  — ЖЁЛТЫЙ
70–100 — КРАСНЫЙ
```

## Что создаёт скрипт

Скрипт:

```text
preparation_data/lsi/script_for_lsi_signals.py
```

Результаты:

```text
data/lsi/results/lsi_signals.xlsx
data/lsi/results/lsi_chart.png
data/lsi/results/lsi_contributions_chart.png
```

В Excel есть листы:

- `lsi_daily` — главная таблица для дашборда;
- `lsi_full` — полная диагностическая таблица;
- `backtest` — проверка на декабре 2014, феврале–марте 2022, августе 2023 и отложенном периоде;
- `sensitivity` — анализ чувствительности весов ±20%;
- `data_quality` — проверка заполненности модулей;
- `method_config` — формула, веса и параметры метода.

## Важное замечание по M5

В текущем файле M5 колонки `mad_score_cbr` и `mad_score_roskazna` пустые, а `flag_budget_drain` везде равен 0. Поэтому M5 пока не влияет на LSI. Это нужно либо исправить в парсере M5, либо честно указать в защите как ограничение текущей версии.

LSI-слой уже готов принимать M5: когда в `m5_treasury_signals.xlsx` появятся реальные MAD-score и флаги, вклад M5 автоматически начнёт считаться.
