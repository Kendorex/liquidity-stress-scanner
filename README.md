# Liquidity Stress Scanner

Проект представляет собой систему раннего выявления стресса ликвидности на российском денежном рынке.

Идея проекта — собрать несколько разных источников данных, посчитать по ним отдельные сигналы напряжения и объединить их в один итоговый индекс **LSI — Liquidity Stress Index**. Индекс принимает значения от 0 до 100 и показывает общий уровень давления на ликвидность.

## Структура проекта

```text
liquidity-stress-scanner/
│
├── data/                  # исходные данные и результаты расчётов
├── parsing/               # скрипты для загрузки данных
├── preparation_data/      # расчёт сигналов по модулям M1–M5 и LSI
├── dashboard/             # Streamlit dashboard
│   └── app.py
│
├── run_all.py             # единый запуск расчётной части
├── requirements.txt       # зависимости проекта
└── README.md
```

## Модули системы

В проекте используется пять модулей.

### M1 — обязательные резервы

Модуль анализирует периоды усреднения обязательных резервов и данные RUONIA.  
Основная идея: если фактические остатки банков сильно выше обычного уровня, это может показывать повышенный спрос на краткосрочную ликвидность.

Итоговые файлы:

```text
data/m1/results/m1_signals.xlsx
data/m1/results/m1_full_result.xlsx
```

### M2 — операции репо Банка России

Модуль анализирует аукционы репо ЦБ.  
Главный показатель — отношение спроса к предложению. Если банки резко увеличивают спрос на репо, это может быть признаком нехватки ликвидности.

Итоговые файлы:

```text
data/m2/results/m2_signals.xlsx
data/m2/results/m2_full_result.xlsx
```

### M3 — аукционы ОФЗ

Модуль анализирует спрос на аукционах ОФЗ.  
Используются показатели спроса и доходности размещения. Для доходности используется proxy-spread, то есть отклонение от собственной исторической нормы, а не полноценная рыночная кривая ОФЗ.

Итоговый файл:

```text
data/m3/result/ofz_auctions_m3_signals.xlsx
```

### M4 — налоговый календарь и сезонность

Модуль учитывает налоговые периоды, конец месяца и конец квартала.  
M4 не считается отдельным стресс-сигналом. Он используется как сезонный множитель, который усиливает итоговый LSI в периоды повышенного календарного давления.

Диапазон сезонного множителя:

```text
1.00–1.25
```

Итоговые файлы:

```text
data/m4/results/m4_signals.xlsx
data/m4/results/m4_full_result.xlsx
```

### M5 — бюджетные средства и казначейство

Модуль анализирует движение бюджетных средств.  
Основной источник — данные ЦБ. Данные Росказны используются как дополнительный индикатор, потому что они заполнены не на всём историческом периоде.

Итоговый файл:

```text
data/m5/result/m5_treasury_signals.xlsx
```

## Итоговый индекс LSI

Итоговый индекс считается на основе сигналов M1, M2, M3 и M5.  
M4 используется как сезонная корректировка.

Общая логика:

```text
base_lsi = взвешенная сумма сигналов M1, M2, M3 и M5
lsi = base_lsi × seasonal_factor
```

Статусы индекса:

```text
0–40    зелёная зона
40–70   жёлтая зона
70–100  красная зона
```

Главный итоговый файл:

```text
data/lsi/results/lsi_signals.xlsx
```

В нём находятся итоговые значения LSI, статусы, вклады модулей, активные флаги и служебные показатели для backtest.

## Запуск расчётов

Перед запуском нужно установить зависимости:

```bash
python -m pip install -r requirements.txt
```

Полный пересчёт сигналов запускается командой:

```bash
python run_all.py
```

Команда последовательно пересчитывает:

```text
M1 → M2 → M3 → M4 → M5 → LSI
```

Важно: `run_all.py` пересчитывает сигналы на основе уже загруженных данных в папке `data/`.  
Скрипты для загрузки и обновления исходных данных лежат отдельно в папке `parsing/`.

## Запуск dashboard

Dashboard запускается командой:

```bash
python -m streamlit run dashboard/app.py
```

В dashboard отображаются:

- текущее значение LSI;
- статус индекса;
- главный вклад в LSI;
- активные модульные флаги;
- график LSI;
- вклад модулей в индекс;
- значения M1, M2, M3 и M5;
- backtest на стрессовых периодах;
- качество и покрытие данных;
- итоговая таблица сигналов.

## Backtest

В проекте проверяется поведение LSI на нескольких стрессовых периодах:

```text
декабрь 2014
февраль–март 2022
январь 2025
```

Индекс лучше реагирует на более поздние периоды, потому что по ним лучше покрытие данных. Для ранних периодов качество данных ниже, поэтому реакция индекса может быть слабее.

## Ограничения проекта

У проекта есть несколько важных ограничений:

- в M3 используется proxy-spread доходности, а не полноценная рыночная кривая ОФЗ;
- в M5 основным источником являются данные ЦБ, а данные Росказны используются дополнительно;
- по ранним годам покрытие данных хуже, поэтому backtest за старые периоды нужно интерпретировать осторожно;
- основной LSI — это интерпретируемая скоринговая модель, а не black-box ML;
- ML-блок может использоваться как дополнительная проверка, но не является основной моделью проекта.

## Как читать итоговый LSI

LSI не показывает точный прогноз кризиса. Его задача — показать, когда по нескольким рыночным и банковским индикаторам одновременно растёт давление на ликвидность.

Если LSI зелёный, это значит, что общий стресс низкий.  
Если жёлтый — есть повышенное напряжение.  
Если красный — несколько модулей одновременно показывают сильный стресс.

Отдельный активный флаг модуля не всегда означает переход всего индекса в жёлтую или красную зону. Например, может быть локальный бюджетный отток по M5, но общий LSI останется зелёным, если остальные модули спокойные.
