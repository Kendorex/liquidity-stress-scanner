# Liquidity Stress Scanner

Проект собирает индикаторы стресса ликвидности по модулям M1–M5 и считает итоговый Liquidity Stress Index (LSI).

## Что уже считается

- **M1** — усреднение обязательных резервов и RUONIA.
- **M2** — спрос на аукционах РЕПО ЦБ.
- **M3** — размещения ОФЗ Минфина. Доходностный спред сейчас используется как `proxy_yield_spread`: отклонение доходности от собственной скользящей медианы, а не от полноценной рыночной кривой ОФЗ.
- **M4** — налоговый календарь. Это не отдельный стресс-сигнал, а сезонный множитель `m4_seasonal_factor`.
- **M5** — средства федерального бюджета/казначейства. Главный источник — данные ЦБ SORS, Росказна используется как дополнительный источник при наличии данных.
- **LSI** — интерпретируемая взвешенная формула на основе `m1_score`, `m2_score`, `m3_score`, `m5_score` и сезонного множителя M4.

## Быстрый запуск

```bash
python run_all.py
```

Команда пересчитывает M1–M5 и затем итоговый LSI.

## Главные итоговые файлы

- `data/m1/results/m1_signals.xlsx`
- `data/m2/results/m2_signals.xlsx`
- `data/m3/result/ofz_auctions_m3_signals.xlsx`
- `data/m4/results/m4_signals.xlsx`
- `data/m5/result/m5_treasury_signals.xlsx`
- `data/lsi/results/lsi_signals.xlsx`

## Единый формат модулей

Каждый модуль теперь отдаёт стандартизированные поля для LSI:

- `date`
- `mX_score`
- `mX_flag`
- `mX_signal`

Для M4 дополнительно используется `m4_seasonal_factor`, потому что M4 работает как календарный множитель, а не как самостоятельный рыночный стресс.

## Важные ограничения

1. В M3 нет полноценной кривой ОФЗ. Используется `proxy_yield_spread`.
2. В M5 Росказна заполнена неравномерно, поэтому главный источник M5 — ЦБ SORS.
3. ML-модель не является главным результатом. Основной LSI — интерпретируемая скоринговая формула.
4. Для backtest добавлен `event_stress_flag` по известным стрессовым окнам, потому что ground truth по ликвидности не всегда совпадает с календарём стрессовых событий.
